# Post Patterns to Watch

A taxonomy for identifying recurring LinkedIn post patterns that may indicate weak signals, hype, vendor positioning, operational pressure, governance needs, content opportunities or low-value noise.

## Purpose

This document defines reusable post-pattern categories for LinkedIn-based trend radar work.

It helps teams, organizations, agencies and AI-assisted workflows interpret recurring post formats, narrative structures and discourse patterns without confusing visibility with evidence.

A post pattern can help answer:

* Why did this post become visible?
* What kind of signal may it represent?
* Is this pattern useful for trend detection?
* Is this pattern mostly noise?
* Does this pattern reveal vendor positioning?
* Does this pattern show operational friction?
* Does this pattern indicate leadership pressure?
* Does this pattern require external evidence?
* Does this pattern create a governance, adoption, vendor or content question?

The purpose is not to identify the best-performing LinkedIn formats.

The purpose is to identify post patterns that are useful for radar interpretation.

## Core idea

A post pattern is not evidence by itself.

It is a recurring form that may reveal how a topic is moving, being framed, commercialized, debated, misunderstood or operationalized.

```txt id="rzp8ia"
post format
→ distribution clue

post pattern
→ interpretation clue

signal value
→ review outcome

external evidence
→ verification layer
```

A carousel, poll, document post, comment thread or viral post may be useful because of what it reveals.

But the format alone does not prove trend relevance.

## What this taxonomy is

This taxonomy is:

* a post-pattern recognition tool
* a radar interpretation layer
* a feed-review support tool
* a source-classification support tool
* a noise-reduction support tool
* a trend-note input structure
* a content-idea source
* a reusable standard for teams and AI-assisted workflows

## What this taxonomy is not

This taxonomy is not:

* a LinkedIn growth playbook
* a content-performance benchmark
* a posting strategy
* a virality formula
* evidence assessment
* maturity classification by itself
* a replacement for source classification
* a replacement for external evidence
* a method for scraping or extracting posts

Post patterns should be used as one classification layer within the signal-detection workflow.

## Post pattern groups

The taxonomy uses six pattern groups.

| Group              | What it detects                                   | Example patterns                                                        |
| ------------------ | ------------------------------------------------- | ----------------------------------------------------------------------- |
| Signal patterns    | Early movement or repeated professional attention | new term, repeated friction, practitioner lessons                       |
| Maturity patterns  | Movement across trend maturity stages             | expert debate, vendor push, management hype, operational pressure       |
| Narrative patterns | How a topic is framed or made understandable      | myth/reality, before/after, “X is not Y”                                |
| Evidence patterns  | Whether the post points beyond itself             | research link, regulation reference, case note, technical demo          |
| Noise patterns     | Low-value or distorted input                      | generic hype, AI slop, engagement bait, viral binary                    |
| Handoff patterns   | What the post should trigger                      | governance question, adoption question, vendor evaluation, content idea |

A single post can match more than one pattern.

Use a primary pattern and optional secondary pattern where useful.

## Signal patterns

Signal patterns indicate that something may be emerging, repeating or becoming operationally relevant.

### 1. New-term pattern

Use when a post introduces or amplifies a new term, phrase, category or label.

Examples:

```txt id="jxvwrb"
agentic workflows
AI-native operating model
sovereign AI stack
model risk ownership
human-in-the-loop governance
```

Potential signal value:

* weak signal
* language shift
* vendor category formation
* expert debate
* management hype

Review questions:

* Who is using the term?
* Is it expert-led, vendor-led, consultant-led or regulator-led?
* Does the term clarify something operational?
* Does it rename an existing concept?
* Is the term spreading across source archetypes?

Possible next actions:

```txt id="kq5h78"
watchlist
terminology note
source diversity review
external evidence task
content idea
```

### 2. Repeated-friction pattern

Use when several posts describe the same implementation difficulty, pain point or workflow problem.

Examples:

```txt id="m7l5lx"

AI outputs fail because review routines are unclear
employees use tools before governance is ready
agent workflows break at permission boundaries
```

Potential signal value:

* emerging pattern
* operational pressure
* governance requirement
* adoption friction

Review questions:

* Which source archetypes report the friction?
* Is the friction technical, organizational, cultural or governance-related?
* Does the friction appear across contexts?
* Does it require workflow redesign or policy review?

Possible next actions:

```txt id="x9piwb"

adoption question
governance question
external evidence task
```

Review questions:

## Noise patterns

Noise patterns should be reduced, weighted carefully or routed only as narrative signals.

### 21. AI-slop / generic thought-leadership pattern

Use when a post appears polished but generic, repetitive, low-context or interchangeable.

Indicators:

```txt id="pya0an"
generic AI statements
no concrete example
no source or evidence
recycled phrases
broad transformation claims
little professional specificity
```

Potential signal value:

* noise
* management hype if repeated by relevant leadership sources
* content-quality warning

Review questions:

* Does the post add expertise, context or evidence?
* Is it interchangeable with many other posts?
* Does it reveal a narrative pattern worth tracking?
* Should it be discarded?

Possible next actions:

```txt id="elsr0z"
discard
noise review
management hype watchlist
content-quality note
```

## Review questions

Before acting on a post pattern, ask:

```txt id="wak8aq"
What pattern is visible?
Which source archetype produced it?
Is the pattern repeated?
Is the pattern source-diverse?
Does it indicate signal, hype, evidence, noise or handoff?
Is external evidence available?
Is the pattern useful for the radar purpose?
Is the next action necessary and proportionate?
Should the observation be kept, reviewed, archived or deleted?
```

## AI-assisted use

An AI assistant may help classify post patterns, source archetypes, signal types, maturity levels and next actions if the input has already been minimized and approved.

The AI assistant should not request, infer or store unnecessary personal data.

The AI assistant should work from source archetypes, minimal signal notes and external evidence whenever possible.

Useful AI-assisted tasks include:

* classifying post patterns from minimized observations
* distinguishing format from signal value
* identifying possible pattern combinations
* suggesting likely signal types
* suggesting evidence needs
* routing patterns to governance, adoption, vendor or content workflows
* identifying repeated patterns across minimized notes
* detecting likely noise patterns

Avoid using AI for:

* extracting LinkedIn posts at scale
* processing raw connection exports without review
* profiling named individuals
* ranking people by influence or trustworthiness
* inferring sensitive attributes
* storing unnecessary identifiers
* treating virality or format as evidence
